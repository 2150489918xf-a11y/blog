## 1 课程设计概述

### 1.1 课程设计名称

**知行成长笔记** — 基于 Supabase BaaS 的个人博客与作品集网站

### 1.2 设计目的与意义

本次课程设计旨在综合运用《Web前端基础》课程所学的 HTML、CSS、JavaScript 技术，设计并实现一个功能完整、界面美观、响应式的个人博客网站。通过本项目，重点掌握以下技术要点：

- **HTML5 语义化标签**：合理使用 `<header>`、`<nav>`、`<main>`、`<article>`、`<section>`、`<aside>`、`<footer>` 等语义标签，提升页面结构的可读性与可维护性。
- **CSS3 高级特性**：运用 Flex 布局、Grid 布局、CSS 变量（Custom Properties）、动画（@keyframes）、毛玻璃效果（backdrop-filter）等现代 CSS 技术实现丰富的视觉效果。
- **JavaScript 交互逻辑**：通过原生 JavaScript 实现动态数据加载、表单验证、DOM 操作、localStorage 缓存、实时时钟等功能，深入理解浏览器端编程。
- **Supabase BaaS 后端服务**：借助 Supabase 提供的 PostgreSQL 数据库、Auth 认证和 Storage 存储服务，实现数据的增删改查、图片上传和管理后台功能，无需搭建独立服务器，降低了项目部署门槛。
- **响应式设计**：通过媒体查询实现桌面端、平板端、移动端的多端适配，确保网站在各设备上均有良好的显示效果。

本项目作为计算机专业学生的个人作品集，不仅用于展示学习成果和课程项目，也可作为未来求职时的个人网站，具有实际应用价值。

### 1.3 设计任务与要求

本设计的具体任务与要求如下：

1. **页面数量要求**：至少包含 10 个以上 HTML 页面，覆盖首页、文章列表、文章详情、项目展示、资源导航、关于页面、联系页面等多个功能模块。
2. **交互功能**：实现轮播图自动/手动切换、表单验证与提交反馈、动态数据加载等交互效果。
3. **响应式布局**：网站需在桌面端、平板端、移动端均有良好的显示效果。
4. **数据管理**：通过 Supabase 后端实现文章、项目、资源链接、学习计划等数据的增删改查，并配备完整的管理后台。
5. **代码规范**：HTML 标签语义化，CSS 命名规范，JavaScript 代码结构清晰、注释完整。
6. **无需构建工具**：整个项目为纯静态站点，无需 Webpack、Vite 等构建工具即可直接在浏览器中运行。

## 3 需求分析

### 3.1 功能需求

本项目需要实现以下功能模块：

#### 3.1.1 页面构成

| 页面 | 文件路径 | 功能描述 |
|------|---------|---------|
| 首页 | `index.html` | 顶部导航、Hero 轮播图、近期更新卡片组、侧边栏目录、全局页脚 |
| 学习笔记库 | `pages/articles/index.html` | 文章分类列表，支持筛选，展示文章标题、分类、发布时间 |
| 文章详情页 | `pages/articles/*.html` | Markdown 渲染文章内容，含封面图、标签、面包屑导航 |
| 实战教程库 | `pages/projects/index.html` | 项目卡片展示，含技术栈标签、状态指示、演示/仓库链接 |
| 项目详情 | `pages/projects/detail.html` | 动态加载项目数据，含项目描述、截图、技术栈详情 |
| 资源导航 | `pages/resources/links.html` | 按分组展示外部资源链接，侧边栏快速跳转 |
| 互动交流 | `pages/about/contact.html` | 联系表单提交、留言板展示（localStorage 持久化） |
| 关于我 | `pages/profile/resume.html` | 个人简介、技能徽章、时间线、联系方式 |
| 个人信息设置 | `pages/profile/settings.html` | 覆盖所有 HTML 表单类型，含前端验证 |
| 学习计划表 | `pages/data/schedule.html` | 可视化学习计划表格，状态指示（待开始/进行中/已完成） |
| 管理后台 | `admin.html` | 完整 CMS：登录认证、文章/项目/资源/导航/设置等模块 CRUD |

**总计 13 个 HTML 页面**，满足课程设计要求。

#### 3.1.2 交互效果

- **Hero 轮播图**：支持自动定时切换（5 秒间隔）和手动点击切换，带 CSS 动画过渡效果。
- **表单验证**：各表单在提交前进行必填检查、格式校验（如邮箱格式、URL 格式），验证失败显示错误提示。
- **实时时钟**：页面右下角显示实时时间（时:分:秒），每秒自动更新。
- **回到顶部按钮**：页面滚动超过 300px 时显示，点击平滑滚动回顶部。
- **标签输入**：在文章/项目编辑表单中支持按回车添加标签、点击 × 删除标签。
- **封面上传**：支持点击选择图片或直接粘贴（Ctrl+V）上传，自动上传至 Supabase Storage。
- **Markdown 编辑器**：在管理后台提供双栏编辑视图，左侧写 Markdown，右侧实时预览渲染效果。
- **管理后台模块切换**：点击侧边栏模块按钮动态切换列表视图，无需刷新页面。
- **Toast 消息提示**：操作成功/失败时在屏幕底部弹出提示，3.5 秒后自动消失。

#### 3.1.3 展示内容

- **文章系统**：标题、URL 别名（slug）、摘要、正文（Markdown）、封面图、分类、标签、阅读时长、发布状态。
- **项目系统**：标题、slug、封面图、摘要、详细介绍（Markdown）、项目状态（进行中/已完成/待开始）、技术栈标签、演示地址、仓库地址、排序权重。
- **资源导航**：资源分组（名称、描述、排序）、分组内的具体链接（标题、URL、角标文字、描述）。
- **学习计划**：任务名称、时间安排、目标说明、当前状态、排序。
- **站点配置**：站点名称、品牌名、作者信息、联系方式、页脚文字、SEO 关键词等全站配置项。
- **简历区块**：个人简介、技能徽章、时间线、自我评价等关于页面内容模块。

### 3.2 非功能需求

- **页面美观**：整体采用珊瑚粉 + 天蓝 + 薰衣草紫的渐变背景，配合毛玻璃（Glassmorphism）卡片设计，风格清新文艺、简约现代，配色协调统一。
- **浏览器兼容性**：使用 CSS 前缀（`-webkit-`、`-moz-`）确保动画和毛玻璃效果在 Chrome、Edge、Firefox、Safari 等主流浏览器中均能正常显示。
- **响应式适配**：通过 CSS 媒体查询实现三档响应式断点：
  - 桌面端（>1024px）：双栏布局，侧边栏导航。
  - 平板端（768px–1024px）：单栏为主，适当缩小间距。
  - 移动端（<768px）：汉堡菜单，全宽布局，隐藏侧边栏。
- **加载速度**：
  - 采用 localStorage 两层缓存机制（10 分钟 TTL），重复访问直接读取本地缓存，减少网络请求。
  - 所有静态资源通过 CDN 加载（Supabase SDK、Font Awesome）。
  - 无需构建工具，减少页面加载等待时间。

### 4.2 设计思路与整体风格

#### 4.2.1 主题定位

**简约文艺清新风**，面向计算机专业学生群体，兼顾技术感和亲和力。整体气质：
- 不使用过度设计，强调内容优先。
- 配色活泼但不刺眼，渐变背景营造年轻感。
- 毛玻璃卡片提供层次感但不喧宾夺主。

#### 4.2.2 配色方案

| 色值 | 用途 | 说明 |
|------|------|------|
| `#eea2a2` | 背景渐变起点 | 珊瑚粉，温暖感 |
| `#57c6e1` | 背景渐变中间 | 天蓝色，清新感 |
| `#b49fda` | 背景渐变终点 | 薰衣草紫，文艺感 |
| `#ffffff` | 卡片/面板背景 | 白色毛玻璃面 |
| `#4c4948` | 主文字颜色 | 深灰，近黑色可读性好 |
| `#858585` | 次要文字颜色 | 中灰 |
| `#49b1f5` | 主强调色 | 天蓝色，用于链接、按钮、高亮 |
| `#ff7242` | 次强调色 | 橙金色，用于徽章、重要提示 |
| `#99a9bf` | 占位符文字 | 浅灰蓝 |

#### 4.2.3 排版布局

- **字体**： `"Segoe UI", "PingFang SC", "Microsoft YaHei", sans-serif`，覆盖 Windows/macOS/iOS/Android 主流字体。
- **布局方式**：CSS Flexbox 为主，配合 CSS Grid 实现网格卡片布局。
- **间距规范**：统一使用 12px 为基准单位的间距系统（8px、12px、16px、24px、32px、48px）。
- **圆角规范**：全站统一 `border-radius: 12px`，卡片、按钮、输入框一脉相承。
- **阴影规范**：轻阴影 `0 3px 8px 6px rgba(7,17,27,0.05)`，hover 时加深至 `0.09` 透明度。

#### 5.2.2 Supabase 数据查询与缓存

```javascript
/**
 * supabase-core.js — 带缓存的通用列表查询
 * ============================================
 * @param {string} table   - 数据库表名
 * @param {string} cacheKey - localStorage 缓存键名
 * @param {number} cacheTime - 缓存有效期（毫秒）
 * @param {object} options - 查询选项（筛选条件、排序字段等）
 *
 * 逻辑：先查 localStorage 缓存 → 命中则直接返回 → 未命中则查 Supabase → 结果写入缓存
 */
async function getCached(table, cacheKey, cacheTime, options) {
  // 第1步：读取 localStorage 缓存
  var cached = readCache(cacheKey, cacheTime);
  if (cached) {
    console.log('[BlogDB] 命中缓存 [' + cacheKey + ']');
    return cached;
  }

  // 第2步：缓存未命中，查询 Supabase 数据库
  console.log('[BlogDB] 缓存未命中，查询数据库 [' + table + ']');
  var data = await listRecords(table, options);

  // 第3步：将查询结果写入缓存，下次访问直接使用
  writeCache(cacheKey, data);
  return data;
}

/**
 * 获取已发布的文章列表（5 分钟缓存）
 */
async function getPublishedArticles() {
  return getCached("articles", "articles_published", 5 * 60 * 1000, {
    filters: [{ column: "published", operator: "eq", value: true }],
    orderBy: "created_at",
    ascending: false,
    emptyValue: null
  });
}
```

#### 5.2.3 轮播图自动切换

```javascript
/**
 * 轮播图初始化与控制
 * ===================
 * 支持：自动定时切换（5秒）、手动点击切换、鼠标悬停暂停
 */
function initCarousel() {
  var carousel = document.querySelector('.carousel');
  if (!carousel) return;

  var slides = carousel.querySelectorAll('.carousel-slide');
  var dots = carousel.querySelectorAll('.carousel-dot');
  var current = 0;
  var timer;

  function goTo(index) {
    // 移除所有 active 类，切换到目标幻灯片
    slides.forEach(function (s) { s.classList.remove('active'); });
    dots.forEach(function (d) { d.classList.remove('active'); });
    slides[index].classList.add('active');
    dots[index].classList.add('active');
    current = index;
  }

  function next() {
    goTo((current + 1) % slides.length);
  }

  // 自动切换：每 5 秒调用一次
  function startAuto() {
    timer = setInterval(next, 5000);
  }

  // 鼠标悬停时暂停自动切换（提升用户体验）
  carousel.addEventListener('mouseenter', function () {
    clearInterval(timer);
  });
  carousel.addEventListener('mouseleave', startAuto);

  // 手动点击指示器切换
  dots.forEach(function (dot, idx) {
    dot.addEventListener('click', function () {
      goTo(idx);
      clearInterval(timer);
      startAuto(); // 重置自动计时器
    });
  });

  startAuto();
}
```

#### 5.2.4 通用管理表单构建器（核心架构）

```javascript
/**
 * admin-form.js — 通用表单 HTML 构建器
 * ========================================
 * 根据 MODULES 配置中的字段定义，自动生成对应的表单 HTML。
 * 支持：text、textarea、select、checkbox、cover、tags、datalist、markdown 等类型。
 *
 * 设计思路：一个 buildFieldHtml() 函数处理所有字段类型，
 * 后续新增字段类型只需在函数中增加 if 分支，无需修改调用方代码。
 */
function buildFieldHtml(field, value, options) {
  if (field.type === 'checkbox') {
    // 复选框特殊处理：label 内嵌 input，不显示额外 label 文字
    return '<div class="form-group' + (field.full ? ' is-full' : '') + '">' +
      '<label style="display:flex;align-items:center;gap:10px;cursor:pointer;">' +
        '<input type="checkbox" name="' + field.name + '" ' +
        (value ? 'checked' : '') + ' style="width:auto;">' +
        '<span>' + esc(field.label) + '</span>' +
      '</label>' +
    '</div>';
  }

  if (field.type === 'select') {
    // 下拉选择框：动态生成 option 列表
    var optionHtml = options.map(function (opt) {
      return '<option value="' + esc(opt.value) + '"' +
        (String(opt.value) === String(value) ? ' selected' : '') + '>' +
        esc(opt.label) + '</option>';
    }).join('');
    return '<div class="form-group' + (field.full ? ' is-full' : '') + '">' +
      '<label>' + esc(field.label) + (field.required ? ' *' : '') + '</label>' +
      '<select name="' + field.name + '"><option value="">请选择</option>' +
      optionHtml + '</select>' +
    '</div>';
  }

  // 更多字段类型（textarea、cover、tags、markdown 等）见完整代码
  // ...

  // 默认：input 标签（text、number、url、email 等）
  return '<div class="form-group' + (field.full ? ' is-full' : '') + '">' +
    '<label>' + esc(field.label) + (field.required ? ' *' : '') + '</label>' +
    '<input type="' + field.type + '" name="' + field.name + '" ' +
    'value="' + esc(value) + '" placeholder="' + esc(field.placeholder || '') + '">' +
  '</div>';
}

/**
 * 通用列表渲染器
 * 根据 MODULES 配置的 columns 定义，动态生成表格列和操作按钮
 */
function renderGenericTable(module, moduleId, records) {
  var header = module.columns.map(function (col) {
    return '<th>' + esc(col.label) + '</th>';
  }).join('');

  var rows = records.map(function (record) {
    var cells = module.columns.map(function (col, idx) {
      // 支持自定义格式化函数（如状态徽章、日期格式化）
      var value = col.format ? col.format(record[col.key], record) : record[col.key];
      var cls = idx === 0 ? ' class="title-cell"' : '';
      // col.html=true 时允许渲染 HTML（如带颜色的状态徽章）
      return '<td' + cls + '>' + (col.html ? value : esc(value)) + '</td>';
    }).join('');

    // 生成操作按钮：编辑 / 切换可见 / 删除
    var actions = '<button onclick="window._moduleEdit(\'' + moduleId + '\',\'' + record.id + '\')"><i class="fas fa-pen"></i></button>';
    actions += '<button class="danger" onclick="window._moduleDel(\'' + moduleId + '\',\'' + record.id + '\')"><i class="fas fa-trash"></i></button>';

    return '<tr>' + cells + '<td><div class="action-group">' + actions + '</div></td></tr>';
  }).join('');

  document.getElementById('moduleListContainer').innerHTML =
    '<table class="article-table"><thead><tr>' + header + '<th>操作</th></tr></thead><tbody>' + rows + '</tbody></table>';
}
```

### 5.3 关键问题与解决方法

#### 5.3.1 浏览器跨域请求与 CDN 加载问题

**问题描述**：部分浏览器或网络环境下，访问 `admin.html` 时 Supabase SDK（通过 CDN 加载）可能未成功加载，导致 JavaScript 数据操作全部失败。

**解决方法**：在 `supabase-core.js` 中使用 `window.supabase` 全局变量判断 SDK 是否加载成功。若未加载，`getClient()` 返回 `null`，所有数据库操作降级为本地缓存模式，避免页面完全崩溃。同时在控制台输出友好警告：
```javascript
if (!window.supabase) {
  console.warn('[BlogDB] Supabase SDK 未加载，请检查 CDN 是否可访问');
  return null;
}
```

#### 5.3.2 图片粘贴上传功能

**问题描述**：在 Markdown 编辑器中，用户期望直接 Ctrl+V 粘贴剪贴板图片并自动上传到 Supabase Storage，而非手动选择文件。

**解决方法**：监听 textarea 的 `paste` 事件，遍历 `clipboardData.items` 找到类型为 `image/*` 的文件，调用 `BlogDB.uploadImage()` 上传，将返回的 URL 以 Markdown 语法 `![](url)` 插入到光标位置：
```javascript
mdInput.addEventListener('paste', function (e) {
  var items = e.clipboardData.items;
  for (var i = 0; i < items.length; i++) {
    if (items[i].type.indexOf('image') !== -1) {
      e.preventDefault();
      var file = items[i].getAsFile();
      BlogDB.uploadImage(file, 'articles').then(function (url) {
        var imgMd = '\n![' + file.name + '](' + url + ')\n';
        // 插入到光标位置
        mdInput.value = mdVal.substring(0, start) + imgMd + mdVal.substring(end);
      });
    }
  }
});
```

#### 5.3.3 JavaScript 文件组织与跨文件调用

**问题描述**：将大型 JS 文件拆分为多个模块后，各模块之间的函数调用依赖全局 `window` 对象传递，存在加载顺序问题。例如 `structured-editor.js` 需要访问 `admin.js` 中定义的 `$moduleForm` DOM 变量。

**解决方法**：
1. 在 `admin.js` 的 IIFE 执行时，主动将需要跨文件共享的 DOM 引用挂载到 `window` 对象上：
   ```javascript
   window.adminModuleForm = document.getElementById('moduleForm');
   ```
2. `structured-editor.js` 通过 `window.adminModuleForm` 获取 DOM 引用，而非依赖变量提升：
   ```javascript
   function getModuleForm() {
     return window.adminModuleForm || document.getElementById('moduleForm');
   }
   ```
3. 严格控制 HTML 中的 `<script>` 加载顺序，确保依赖关系正确：
   ```
   supabase-core.js → supabase.js → admin-modules.js → admin-form.js → structured-editor.js → admin.js
   ```

#### 5.3.4 localStorage 缓存策略

**问题描述**：直接查询 Supabase 数据库会产生网络延迟，影响页面首次加载速度；且频繁请求会增加 API 调用量。

**解决方法**：实现两级缓存策略：
1. **内存缓存**（data-layer.js）：模块级变量缓存当前会话中已解析的数据。
2. **localStorage 缓存**（supabase-core.js）：设置 5~10 分钟 TTL，存储查询结果。
3. **写入失效**：所有创建/更新/删除操作后自动调用 `clearContentCache()` 清除缓存，确保下次读取拿到最新数据。

### 6.2 收获与知识点总结

通过本次课程设计，我对 Web 前端开发有了更深入的理解，具体收获如下：

1. **CSS 进阶特性**：掌握了 CSS 变量（Custom Properties）进行主题管理的用法；深入理解了 Flex 布局和 Grid 布局的适用场景；学会了使用 `backdrop-filter` 实现毛玻璃视觉效果，以及 CSS `@keyframes` 创建流畅动画。
2. **JavaScript 模块化思维**：通过将 1500+ 行的 `script.js` 拆分为 `data-layer.js`、`page-sections.js`、`profile-renderer.js`、`script.js` 四个文件，我实践了"关注点分离"原则。每个文件职责单一，代码可读性和可维护性大幅提升。
3. **Supabase BaaS 实践**：通过实际项目掌握了 PostgreSQL 数据库设计（表关系）、Auth 认证流程、Storage 文件上传等操作，深入理解了"后端即服务"模式在个人项目中的应用。
4. **缓存策略设计**：实现了 localStorage 两级缓存（内存 + 持久化），在减少网络请求的同时保证了数据一致性。
5. **前端表单系统设计**：通过构建通用表单生成器（`buildFieldHtml`）和通用列表渲染器（`renderGenericTable`），实现了 CMS 的高度可扩展性——新增一个管理模块只需在 `MODULES` 配置对象中添加定义，无需编写重复的 HTML 和 JavaScript。

### 6.3 存在不足与改进方向

1. **图片懒加载未实现**：目前所有图片在页面加载时同时请求，大图列表页可能有性能问题。后续可引入图片懒加载（Intersection Observer API）优化首屏加载速度。
2. **搜索功能缺失**：目前文章和项目只能通过分类筛选，缺少全文搜索功能。可考虑引入 Algolia 或 Supabase 的全文搜索能力。
3. **Markdown 渲染功能有限**：当前 Markdown 预览仅支持基础语法（标题、加粗、链接、图片、列表、引用），代码高亮、表格等未支持。可引入 `marked.js` 等成熟库替换现有简单渲染逻辑。
4. **未使用构建工具**：项目全程使用纯静态文件，开发体验上不如 Vite/Webpack 下的热模块替换（HMR）。后续可引入 Vite 构建工具，提升开发效率。

### 6.4 课程学习心得

《Web前端基础》这门课程让我从一个只会复制粘贴 HTML/CSS 代码的学习者，逐步成长为能够独立设计并实现一个完整 Web 应用的学习者。从最初的手写静态页面，到理解 CSS 布局原理，再到运用 JavaScript 实现复杂交互，最后借助 Supabase 构建完整的数据驱动网站——每一步都是宝贵的积累。

本次课程设计让我深刻体会到：**前端开发不仅是写代码，更是解决实际问题**。如何组织代码结构、如何设计可扩展的系统、如何处理用户输入和异常情况，这些"软技能"往往比掌握某个 API 更加重要。

未来，我计划继续深化前端技术栈的学习，重点关注 TypeScript（类型安全）、Vue3/React（组件化开发）、Vite（构建工具）以及 Web 性能优化等方向，逐步向专业前端工程师的标准靠拢。
